1.2 页面清单与职责

💬 签到对话 /chat

/chat · 底部导航 Tab 1 · 默认首页

核心页面。连续对话模式，消息按日期分组显示。每天 AI 主动问候并引用昨天话题。

  • 消息输入 + AI 回复气泡
  • MBTI 跳过提示横幅
  • 情绪标签弹层
  • 洞察卡片 + 微行动推荐

🧰 心理工具箱 /toolkit

/toolkit · 底部导航 Tab 2

6 个结构化心理自助工具的入口页。根据用户当前情绪和 MBTI 类型智能推荐排序。

  • 工具卡片列表（6 个）
  • 智能推荐标记
  • 各工具独立子页面

📊 情绪记录 /history

/history · 底部导航 Tab 3

纵向情绪追踪与数据可视化。30 天日历视图 + 情绪分布统计 + 连续签到。

  • 月度日历（颜色映射情绪）
  • 情绪饼图/分布
  • 签到天数与连续记录

⚙️ 设置 /settings

/settings · 底部导航 Tab 4

个人信息、MBTI 重新选择、AI 记忆管理（查看/删除记忆条目）、通知偏好、退出登录。

  • MBTI 重测入口
  • AI 记忆条目管理
  • 隐私与协议链接

/login · 公开页面

邮箱 + 密码登录/注册。Supabase Auth 驱动。登录成功后检查是否首次，首次进入 MBTI 引导。

4.2 核心数据流

数据流路径触发时机数据格式
用户消息 → AI 回复 前端 → FastAPI → DeepSeek V3 → FastAPI → 前端 用户发送消息 SSE 流式输出
情绪识别 对话内容 → Prompt 策略 → DeepSeek → 情绪标签 AI 判断对话充分时 JSON {emotion, sub_emotion, confidence}
MBTI 风格注入 用户 MBTI → Prompt 模板选择 → System Prompt 拼装 每次对话开始 System Prompt 文本
记忆读写 前端 → FastAPI → MySQL memories 表 对话结束时提取 / 下次对话时注入 JSON {key, content, importance}
情绪记录 签到数据 → FastAPI → MySQL checkin_logs 表 用户确认情绪后 JSON {date, emotion, insight, action}
危机干预 对话内容 → 关键词检测 → 强制安全响应 检测到自伤/自杀关键词 强制中断 + 热线信息

4.3 MBTI 人格映射数据流

16 型 MBTI → 4 种 AI 风格的映射规则：

T/F 维度决定回应框架：T（思考型）= 逻辑分析优先 · F（情感型）= 共情支持优先
E/I 维度决定互动模式：E（外向型）= 主动引导 · I（内向型）= 温和陪伴

FI（默认）温暖细腻型 → INFP/INFJ/ISFP/ISFJ
FE 活力共情型 → ENFP/ENFJ/ESFP/ESFJ
TI 冷静分析型 → INTP/INTJ/ISTP/ISTJ
TE 理性引导型 → ENTP/ENTJ/ESTP/ESTJ
Part 05

页面状态与对话状态机

5.1 对话状态机

签到对话的 AI 行为由状态机驱动，不同阶段触发不同的 Prompt 策略。

状态触发条件AI 行为可转移到
GREETING 进入聊天页 发送时段问候语（早/午/晚/深夜） LISTENING
LISTENING 用户开始倾诉 共情回应 + 适度追问（MBTI 风格） DETECTING / CRISIS
DETECTING 对话 ≥3 轮 且 AI 已识别情绪 推送情绪标签，请求用户确认 INSIGHT
INSIGHT 用户确认情绪 生成洞察卡片 + 微行动推荐 COMPLETE
COMPLETE 洞察已展示 推荐工具箱 → 写入签到记录 GREETING（下次）
CRISIS 检测到危机关键词 强制安全响应 + 热线 + 安全计划 LISTENING（降级后）

6.3 架构演进规划

随着版本迭代，系统架构需要相应演进以支撑新功能和更大规模。

V1.0 → V1.5 架构变更

体验增强期
  • 新增 Redis 缓存层：加速 AI 记忆读取
  • 新增推送服务：接入 FCM/APNs 或 UniPush
  • 新增 ASR 模块：接入语音识别 API
  • 数据库索引优化：情绪趋势查询加索引
  • 前端 PWA 化：离线缓存 + 添加到桌面

V1.5 → V2.0 架构变更

社交与变现期
  • 新增支付服务：接入微信/支付宝支付
  • 新增社区模块：树洞内容存储 + Feed 流
  • 新增 OAuth 集成：微信/手机号登录
  • CDN 静态资源：音频/图片资源分发
  • 数据库读写分离：应对社区读流量
  • PDF 报告生成服务：后台异步渲染

V2.0 → V3.0 架构变更

平台化期
  • 微服务拆分：对话/工具/社区/支付独立服务
  • API 网关：统一鉴权 + 限流 + 计费
  • 多模态 AI 管线：图像理解 + 语音分析
  • 数据仓库：企业报告用大数据分析
  • 多租户架构：支撑 B2B 企业隔离
  • 国际化 i18n：前后端多语言
